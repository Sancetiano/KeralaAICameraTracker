## 3. How to Contribute
I welcome bug fixes and feature suggestions!

### Reporting Bugs/ Feature Suggestions
1. Check the Issues tab to see if the bug has already been reported or the feature suggestion has already been made.
2. If not, open a new issue with clear steps to reproduce the behavior.
3. Since the original maintainer is inactive, please do not report issues to the upstream repository; keep all discussions here.

### Submitting Changes
1. **Fork** the repository to your own account (for development purposes only).
2. Create a feature branch for your fix. (e.g., git checkout -b feat/new-ui-feature)
3. Commit your changes with clear, descriptive messages of the features/ fixes made. The more the detail, the more easier to read and understand.
4. Submit a **Pull Request** to my `main` branch.
5. Ensure your code follows my existing style and includes comments where necessary.

## 4. Code Standards
1. Maintain the existing code style
2. Add comments for any complex logic, especially if it interfaces with the original "Base Code."
3. Ensure that any new dependencies you introduce are compatible with our hybrid licensing model.
